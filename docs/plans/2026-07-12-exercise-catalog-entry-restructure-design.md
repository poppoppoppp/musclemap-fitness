# 动作数据与入口结构调整设计

## 目标

将内置动作从 48 个扩充到 260 个，移除底部“动作库”入口，并把现有动作库页面改造为“我的 → 动作管理”二级页面。现有动作详情、动作抽屉、肌群筛选和加入训练流程保持兼容。

## 现状

- `src/data/exercises.ts` 是动作数据唯一公共出口，当前包含 48 个动作。
- `Exercise` 类型已具备中英文名、主次肌群、器械、分类、标签及教学字段。
- `ExercisePickerSheet` 通过 `src/utils/exerciseFilters.ts` 读取统一动作出口，并在组件外预构建肌群索引。
- `/exercises` 同时服务动作库、动作详情返回路径和训练模板入口，因此不能删除整条路由链。
- “我的”页面已有统一的 `EntryRow` 菜单模式；二级页面保留底部导航并高亮所属一级入口。

## 方案

保留现有 48 个动作定义，在 `src/data/exerciseCatalog/` 下按部位新增 212 个动作，通过 `src/data/exercises.ts` 聚合为唯一的 260 项出口。使用轻量构造器提供空教学数组等默认值，不改变公共 `Exercise` 类型。

新增前臂屈肌、前臂伸肌、髋内收肌、髋外展肌注册项，使新增动作的所有肌群 ID 都能被现有注册表验证。它们只参与文字展示、搜索和分类，不修改 2D/3D 肌肉模型。

复用 `/exercises` 和 `ExerciseLibrary` 作为“动作管理”，增加返回“我的”、总数及当前结果数；保留动作详情路由。底部导航删除动作库项并改为三列，访问 `/exercises` 时高亮“我的”。

## 数据与搜索

- 最终动作数严格为 260，原有 48 个 ID 全部保留。
- 新动作覆盖胸、背、肩、肱二头肌、肱三头肌、前臂、核心、臀、股四头肌、腘绳肌、小腿、内收肌、外展肌和全身复合动作。
- 器械选项扩充至史密斯机、壶铃、双杠、地雷架、健身球、哈克深蹲机等商业健身房常见器械。
- 别名继续使用现有 `tags` 字段；抽屉搜索继续匹配名称、英文名、标签、器械和肌群名称。
- 260 项规模继续使用 `useMemo` 加线性过滤，不引入虚拟列表或新依赖。

## 路由与交互

- `/exercises`：动作管理列表，底部“我的”高亮。
- `/exercises/:exerciseId`：保留现有动作详情行为。
- “我的”训练管理区新增“动作管理”菜单项。
- 动作管理返回按钮固定返回 `/data-management`。
- 保留 Vercel SPA rewrite 与现有 PWA 设置；仓库没有 Electron 运行时，无额外 Electron 改动。

## 校验与测试

新增 Playwright 数据测试检查总数、原数据保留、唯一性、必填项、肌群注册、合法器械和合法枚举；同时验证各目标部位覆盖。端到端测试覆盖三项底部导航、动作管理进入/返回/刷新/搜索/筛选、抽屉的新动作搜索和加入训练，以及 390px 移动端无横向溢出。

最终运行 `npm test` 与 `npm run build`。按用户要求不提交或推送 Git。
