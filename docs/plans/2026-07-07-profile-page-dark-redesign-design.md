# “我的”页暗黑重构设计

## 目标

仅重构 `/data-management` 与全局底部导航，使“我的”页延续首页的黑绿运动科技视觉，同时保留现有备份能力，并将身体快照纳入备份。

## 页面结构

- 页面独立使用深黑墨绿背景、荧光绿强调色和克制的玻璃质感，不改变其他业务页面主题。
- 顶部为“我的”标题和圆形用户按钮。
- Hero 卡仅展示训练次数、当前体重、当前腰围；训练次数读取 `readWorkoutLogs().length`，身体指标读取最新有效身体快照，没有数据时显示“未记录”。
- “记录身体数据”、动作进步、身体变化和偏好设置暂时显示页面内状态提示。
- 训练模板和训练历史分别链接到 `/plan-builder` 与 `/workout-history`。
- 数据备份入口展开现有导出、校验、摘要、确认覆盖导入和状态区域。

## 导航

底部导航固定为：首页 `/`、记录 `/workout-log`、动作库 `/exercises`、我的 `/data-management`。导航保持悬浮 pill，所有页面使用暗色材质；当前项使用深绿色高亮块、荧光绿图标和文字。

## 身体数据与备份

新增 `BodySnapshot` 类型和独立 localStorage key。快照按 `date`、再按 `createdAt` 取最新值。备份格式升级到 v2，并包含 `bodySnapshots`。校验器接受 v1 和 v2：v1 导入时将身体快照规范化为空数组；v2 严格校验身体快照。恢复时覆盖身体快照存储。

## 验证

- 先增加失败测试，覆盖导航、页面结构、空状态、最新身体快照、占位提示、备份 v1 兼容和 v2 身体数据往返。
- 在 390×844 下验证无横向滚动、底部导航不遮挡内容。
- 最后运行 `npm run build` 与 `npm run test:e2e`，并进行浏览器截图检查。
