# 成长数据闭环设计

## 目标

将成长页从展示原型升级为完全由用户本地真实记录驱动的功能，补齐训练趋势、身体数据和进度照片的记录、查看、编辑、删除与对比流程，并移除所有生产 Mock 和假入口。

## 数据架构

### 训练与动作

- 训练日志继续作为训练成长的唯一事实来源。
- `Exercise` 增加 `weightType`：`external_weight`、`bodyweight`、`bodyweight_added`、`bodyweight_assisted`。
- 目录工厂按显式字段生成重量类型，已知负重和助力动作明确覆盖。
- 旧纯自重训练中的通用 `weight` 不参与有效重量计算，不猜测历史语义。
- `bodyweight_added` 将组内 `weight` 解释为额外负重；`bodyweight_assisted` 将其解释为助力重量。

### 身体数据

- 身体记录升级为 `BodyMetricRecord`：日期、体重、腰围、臂围、创建时间和更新时间。
- 继续兼容 `musclemap.bodySnapshots.v0.1` 和旧字段 `bodyWeightKg/waistCm`，读取时规范化，首次写入时升级。
- `bodyMetricRepository` 统一负责读取、按日 upsert、更新、删除、排序和验证。
- 同一天只保留一条记录，补录指标时合并原值。

### 进度照片

- 新增 IndexedDB 数据库，元数据和 Blob 使用独立 object store。
- `progressPhotoRepository` 负责创建、读取、更新、删除、分组和 Blob URL 生命周期之外的数据访问。
- 删除操作在事务中同时删除元数据和 Blob。
- 图片元数据保存单一分类、日期、备注、宽高、方向和时间戳，为后续回放保留稳定序列。
- 首次保存后只提示一次“照片当前仅保存在此设备”。

## 真实数据派生

### 时间范围

- 近 4 周、近 3 个月、近 6 个月和全部共用一个状态。
- 有限范围同时计算当前周期与前一等长周期；全部不显示周期比较。
- 成长概览、力量趋势、训练分布和身体趋势都使用同一范围过滤结果。

### 成长概览

- 完成训练、活跃周、平均每周训练均来自真实训练日志。
- 空数据展示空状态和 0，不显示虚假增长。

### 力量趋势

- 动作选择器只包含范围内真实出现过的动作，按最后训练时间倒序。
- 外部负重动作每次训练取最高有效重量，并保留次数和组信息。
- 自重动作按训练日期匹配绝对时间差最小的体重记录；距离相同时优先训练前记录。
- 纯自重有效重量为体重；负重自重为体重加额外负重；助力自重为 `max(体重-助力, 0)`。
- 0 点、1 点和 2 点以上分别展示无数据、单点提示和真实趋势。

### 训练分布

- 主卡和详情使用同一派生结构。
- 详情按肌群列出总有效组数和每个动作贡献组数。

## 页面与交互

### 通用 SnapBottomSheet

- Portal、遮罩、焦点陷阱、Escape、拖拽条和两个吸附点。
- Pointer Events 处理上下拖动和吸附。
- 输入聚焦时自动展开；使用 `visualViewport` 与安全区保持输入和固定保存区可见。
- dirty 表单关闭前统一确认放弃。

### 身体数据

- 成长卡右上角提供“记录数据”，有记录时提供“查看记录”。
- `BodyMetricSheet` 用于新增与编辑，并提供简短测量说明。
- `/growth/body-records` 按月份展示卡片，支持真实编辑与二次确认删除。
- 空状态提供首条记录入口，不显示趋势占位。

### 进度照片

- 成长卡提供“添加照片”，有照片时提供“查看全部”。
- `ProgressPhotoSheet` 支持面部、整体身材和局部肌群单分类选择。
- 局部分类复用现有 2D 人体图，通过映射层合并为稳定照片类别。
- `/growth/photos` 按日期分组并提供全部、面部、整体、局部筛选。
- 图片详情支持查看、编辑元数据和删除。
- `/growth/photos/compare/:category` 在同类至少两张时默认比较最早与最新，也可更换日期。

### 入口治理

- 所有查看、添加、记录、箭头和菜单入口必须连接真实交互。
- 暂未实现的成长回放只打开说明弹层，不进入假页面，不生成视频。
- 无数据时不显示虚假照片、日期、纪录数或趋势。

## 错误处理

- repository 将存储异常转换为用户可理解的结果。
- 图片保存失败时事务回滚，不产生孤立 Blob 或元数据。
- 保存中禁用重复提交。
- 身体数据校验集中管理，至少一项、非负数和宽松合理范围。
- Blob 预览 URL 由 UI 创建并及时撤销。

## 验证

- 纯函数测试覆盖周期边界、前周期、真实动作选择、负重类型和体重最近匹配。
- repository 测试覆盖身体同日合并、迁移、编辑、删除和 IndexedDB 图片 CRUD。
- 页面测试覆盖抽屉、空状态、历史分组、照片单分类、管理和对比。
- 390×844 验证横向溢出、拖拽吸附和键盘可见性。
- 最终运行生产构建、完整 Playwright 和 `git diff --check`。
