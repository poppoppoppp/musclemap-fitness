# 体态改善功能闭环设计

## 目标

将现有的一次性体态动作包浏览器升级为本地优先的周期改善功能。用户可以完成非诊断式初筛，获得可解释的合格方案推荐，创建 2 至 4 周计划，执行今日任务，查看完成进度并在周期结束后复测。

本阶段优先完善功能和数据闭环，只进行支撑新功能所必需的界面调整。动画与视觉风格将在后续参考样例确定后单独实施。

## 信息架构

体态改善归属于“成长”，使用独立路由 `/growth/posture`，并作为成长页第三个分类。

- “成长”负责初筛、计划创建、周期进度、反馈和复测。
- “首页”只在存在进行中计划时展示“今日体态任务”，提供快速开始或继续入口。
- “训练记录”负责实际执行和归档，不承担计划管理。
- 现有“添加动作 → 体态改善”保留为次级入口，后续改为从计划添加或跳转创建计划。
- 底部导航保持首页、记录、成长、我的四项，不新增第五个入口。

## 用户流程

1. 用户进入“成长 → 体态改善”。
2. 用户完成非诊断式初筛：
   - 主要目标：舒适度、活动能力、训练表现或外观关注。
   - 关注区域与主观症状。
   - 症状持续时间和强度。
   - 麻木、放射痛、眩晕、胸痛等安全检查。
   - 可用器械、单次时间和每周可训练次数。
   - 可重复填写的基线功能指标。
3. 命中安全风险时停止自动推荐，展示触发项并建议寻求专业评估。
4. 通过初筛后，系统只展示最多两个满足发布门槛的方案，并解释推荐原因、预计时长、适用边界和数据质量。
5. 用户创建 2 至 4 周计划，选择每周次数和训练日。
6. 系统根据计划和已有训练记录计算今日任务。
7. 用户开始今日任务，复用现有训练记录并记录体态计划上下文。
8. 每次训练记录训练前后不适度、难度、完成情况和中止原因。
9. 周期结束后，用户使用相同指标复测并选择继续、调整或结束计划。

首批不实现照片诊断、自动姿态识别、AI 医疗判断和复杂算法推荐。推荐规则必须确定、可解释并可测试。

## 数据模型

### PostureAssessment

保存初筛和复测数据：

- ID、创建时间和记录类型（初筛或复测）。
- 主要目标、关注区域和主观症状。
- 症状持续时间与主观强度。
- 安全检查答案和是否阻断推荐。
- 可用器械、单次时长和每周可训练次数。
- 基线功能指标。

### PosturePlan

保存周期计划：

- ID、当前方案 ID 和状态（进行中、暂停、已完成）。
- 开始日期、周期周数、每周次数和用户选择的训练日。
- 创建计划时的推荐原因和方案质量快照。
- 关联的初筛 ID 与复测 ID。
- 暂停时间和完成时间。

系统不预生成全部日程记录。今日任务、完成率和遗漏次数根据计划规则、当前日期和带计划上下文的训练记录动态计算，避免重复状态和同步问题。

### PostureSessionFeedback

保存单次执行反馈：

- 计划 ID、训练记录 ID和创建时间。
- 训练前后不适度。
- 主观难度与完成状态。
- 中止原因和备注。

### 训练计划上下文

`ActiveWorkout` 和 `WorkoutLog` 增加可选字段：

```ts
posturePlanContext?: {
  planId: string;
  weekIndex: number;
  scheduledDate: string;
}
```

从今日任务开始时，根据计划解析体态方案，创建或追加现有体态方案快照。训练归档后，带 `planId` 的记录自动计入周期进度。普通训练、手动添加体态动作和旧历史数据无需迁移。

## 发布门槛与推荐规则

自动推荐只允许满足以下条件的方案：

- `visibility` 不是 `secondary` 或 `internal`。
- `completeness` 不是 `limited`。
- `dataQuality` 不低于 `medium`。
- 所有必做动作存在可执行剂量。
- 所有必做动作都不存在未解决的图示复核。
- 方案存在可追溯来源。

不满足门槛的方案继续保留在数据集中，但不能进入自动推荐。`adjunct` 方案只能作为补充，不能单独生成完整计划。

推荐规则根据初筛的区域、目标、症状、器械、时间和频率匹配合格方案。结果最多两个，并返回明确的匹配原因。相同输入必须产生相同结果。

## 页面与状态

### 无计划状态

- 功能边界说明。
- “开始初筛”主操作。
- 历史计划入口。

### 初筛流程

- 目标与区域。
- 症状和安全检查。
- 时间、频率与器械。
- 基线评分。
- 中途退出时保存草稿。

### 推荐结果

- 最多两个合格方案。
- 推荐原因、预计时长、每周频率、数据质量和适用边界。
- 创建改善计划操作。

### 进行中计划

- 当前周数、完成率和遗漏次数。
- 今日任务和下次安排。
- 最近训练反馈。
- 暂停、结束和重新评估操作。

### 周期复测

- 复用初筛中的基线指标。
- 展示前后变化。
- 继续、调整或结束计划。

## 执行体验

首批继续复用现有训练页面和体态方案快照。体态动作执行需要补充：

- 根据可信处方预填组数、次数或时长。
- 支持持续时间记录和左右侧提示。
- 在训练中显示方案剂量、专项提示和停止条件。
- 允许记录训练前后不适度和主观难度。
- 允许立即中止并保存原因。

本阶段不设计复杂动画，也不重做整套训练页视觉。

## 异常处理

- 命中安全风险：不生成推荐，显示触发项并建议专业评估。
- 没有合格方案：不降级使用低质量方案，显示当前无合适推荐。
- 已有普通训练：将今日任务追加到当前训练，并明确反馈。
- 已有进行中体态计划：要求先暂停或结束旧计划。
- 数据集更新导致方案不可用：暂停计划并要求重新选择，旧历史继续使用快照。
- 初筛中途退出：保存草稿，下次恢复。
- 单次训练明显不适：允许中止并记录原因，本次不计为完整完成。
- 本地存储数据无效：忽略无效记录并回到安全空状态，不影响普通训练数据。

## 测试与验收

### 领域测试

- 不合格方案不能进入自动推荐。
- `adjunct` 不能单独生成计划。
- 推荐结果确定且包含匹配原因。
- 安全风险阻断推荐。
- 同一时间最多一个进行中计划。
- 今日任务、完成率和遗漏次数计算正确。
- 暂停期间不产生遗漏次数。
- 方案失效后计划暂停，历史快照保持可读。
- 旧训练、旧体态快照和现有备份继续兼容。

### 浏览器流程

- 从成长页完成初筛并创建计划。
- 无风险用户看到最多两个合格推荐。
- 有风险用户看不到创建按钮。
- 首页出现今日任务并能开始训练。
- 有进行中训练时任务正确追加。
- 完成训练后计划进度更新。
- 中止训练保存原因且不计完整完成。
- 周期结束后完成复测并继续或结束计划。
- 320px 和 390px 下无横向溢出，键盘焦点和表单错误可用。

### 完成标准

- 用户可以从初筛走到创建计划。
- 用户可以连续执行多次并查看真实进度。
- 用户可以完成周期复测。
- 不合格内容不会被自动推荐。
- 不新增底部导航，不破坏普通训练流程。
- 相关测试、全量测试、构建和 `git diff --check` 全部通过。
